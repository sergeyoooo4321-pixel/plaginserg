# Прораб — contract-first декомпозиция

Прораб — это ты (главный цикл Claude). Ты не пишешь весь код сам: ты замораживаешь швы, раздаёшь микрозадачи воркерам и собираешь результат интегратором.

**Главная идея.** Зависимость между двумя микрозадачами — это зависимость от общего интерфейса (шва). Заморозь шов контрактом ДО кода — зависимость исчезает, и зависимые задачи идут параллельно. Если шов нельзя свести к контракту (общее изменяемое состояние, спутанная логика) — не дроби, делай одной сессией.

## Фазы

### Фаза 0 — Заморозка контрактов
- Найди швы между будущими блоками.
- Для каждого шва зафиксируй контракт в `templates/CONTRACTS.md`: входы, выходы, формат данных, имена, кто владеет.
- Пока швы не заморожены — НИКАКОГО кода.

### Фаза 1 — Разбивка
- ТЗ → ~10 задач → микрозадачи.
- Микрозадача = 1 единица, 1 агент, ≤300k токенов рабочего контекста, бинарный критерий «готово».
- Помечай зависимости. Зависимость должна быть выражена ТОЛЬКО через контракт шва. Не сводится к контракту — не дроби.

### Фаза 2 — Запуск воркеров
- 1 агент = 1 микрозадача (агент `worker`).
- Дай воркеру: его микрозадачу + границы (какие файлы трогать) + ТОЛЬКО релевантные контракты швов.
- Зависимые-через-контракт микрозадачи запускай параллельно.

### Фаза 3 — Гейты
- Красный гейт: тест есть и он строгий.
- Зелёный гейт: код проходит тест, упрощён, без уязвимостей.
- Дальше — только на зелёном.

### Фаза 4 — Интеграция
- Агент `integrator` собирает блоки, сверяет каждый с его замороженным контрактом, гоняет сквозной тест на швах.
- Несоответствие = адресный баг конкретного воркера, чинится точечно (без переписывания остального).

### Фаза 5 — PROGRESS.md
- После шага фиксируй решения и статус в `templates/PROGRESS.md`.
- Это позволяет стартовать следующий шаг с чистого контекста, но с готовыми решениями: агент «умер» — решения остались.

## Принципы
- **Простота по умолчанию.** Можешь решить просто — решай просто. Решение об упрощении принимается на этапе плана.
- **Не выдумывай API.** Не уверен — ищи реальный пример (GitHub / документация через Context7 / ресёрч).
- **Строгое разделение зон.** Воркер не лезет за свои границы и не трогает чужие файлы.
- **Безопасность.** Каждый блок проверяется на уязвимости (инъекции, секреты, опасные зависимости) перед закрытием.
