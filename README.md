# plaginserg

Плагин для Claude Code: «Прораб» — оркестратор contract-first декомпозиции задач.

## Проблема

Когда большую задачу дробят на части и раздают разным ИИ-сессиям, всё ломается на **швах** — в местах, где части стыкуются друг с другом. Одна микрозадача ждёт интерфейс другой, договорённости расходятся, куски не собираются вместе. Чем больше зависимостей между задачами, тем хуже параллелится работа и тем чаще итог не стыкуется.

## Идея решения

Зависимость между микрозадачами — это на самом деле зависимость от общего **шва** (интерфейса).

Заморозь шов **контрактом ДО кода** — и зависимость исчезает. Теперь даже зависимые микрозадачи идут параллельно: каждый воркер опирается не на чужой код, а на заранее зафиксированный контракт.

**Правило:** если шов нельзя свести к контракту (общее изменяемое состояние, спутанная логика, которую не разрезать чистым интерфейсом) → **не дробить**. Делать одной сессией.

## Как работает (фазы 0→5)

- **Фаза 0 — Заморозка контрактов.** Находим швы между будущими частями и фиксируем их контрактами в `CONTRACTS.md` до написания кода.
- **Фаза 1 — Разбивка.** ТЗ → ~10 задач → микрозадачи (1 микрозадача = 1 агент, ≤300k токенов контекста).
- **Фаза 2 — Запуск воркеров.** Каждый воркер делает свою микрозадачу, опираясь только на контракт шва.
- **Фаза 3 — Гейты.** Проверки качества: красный → зелёный. Пока гейт красный — задача не закрыта.
- **Фаза 4 — Интеграция.** Интегратор собирает результаты воркеров по контрактам.
- **Фаза 5 — PROGRESS.md.** Чеклист фаз обновляется и переживает сброс контекста.

Прораб (главный цикл Claude, то есть сам Claude) ведёт этот процесс сверху.

## Установка

Как плагин Claude Code:

- Положить папку `plaginserg` в каталог плагинов Claude Code, **или** подключить через marketplace.
- Запуск: `/prorab <ТЗ>` — передаёшь техзадание, дальше Прораб ведёт фазы 0→5.

## Про «≤300k контекста»

Это **рабочее правило-ориентир** для агентов: держать одну микрозадачу в пределах ~300k токенов, чтобы агент не терял нить и не упирался в раздутый контекст. Это **не** жёсткий технический лимит рантайма — реальные ограничения зависят от модели и окружения. Цифра нужна как ориентир при разбивке, а не как гарантия движка.
